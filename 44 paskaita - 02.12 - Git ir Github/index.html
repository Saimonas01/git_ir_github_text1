Git ir Github

Git'as - kodo versijavimo programa.
GitHub'as - versijuoto kodo talpykla internete.

Git komandos

  • init - sukurti git repozitoriją dabartiniame aplankale
  • status - pateikia git repozitorijos dabartinę informaciją
  • add - prideda nurodytus failus prie commit'inmui paruoštų failų (stage'ina)
  • rm --cached || restore --staged - išima nurodytus failus ir stage'inimo zonos (unstage'ina)
  • commit -m "žinutė apie commit'ą" - sukuria kodo versiją iš stage'intų failų ir prideda žinutę apie pakitimus (žinutė turėtų būti trumpa ir aiški)
  • restore - sugrąžina nurodytą/us failą/us į pradinę būseną (jeigu tie failai nebuvo add'inti į stage'ingo zoną)
  • push - nusiunčia lokalios repozitorijos commit'us į išorinę repozitoriją
  • pull - atsiunčia išorinės repozitorijos commit'us į lokalią repozitoriją
  • log - išveda commit'ų sąrašą
  • clone - nuklonuoja nurodytą projektą į dabartinį aplankalą
  • checkout commitID|branchName - pereinama į nurodytą kodo versiją arba kitą šaką
  • branch naujosSakosPavadinimas - sukuriama nauja šaka
  • ...

Branch protection rules

Vienas svarbiausių šakų apsaugos taisyklių yra "Pull Request" (bandymas vienos šakos commit'us sukelti į kitą šaką (merge)).
Pull Request taisklės viduje dar reikėtų suteikti "Require approvals" nustatymą, kuris reikalauja, kad nurodytas kiekis kitų projekto dalyvių patvirtintų tą Pull Request'ą, norint, kad šakos būtų apjungtos.

Eiliškumas Pull Request

Apsiemiau daryti kazkokia uzduoti. Kokie mano tolimesnis veiksmai?

  1. Susikurti nauja saka - git brancha naujaSaka
  2. Nueiti i susikurta saka - git checkout naujaSaka
  3. Atlieki uzduoti, kuria apsiemei daryti. Su vienu arba keliais commit'ais
    1. Add'ini pakitimus - git add .
    2. Commit'ini pakitimus - git commit -m 'zinute apie pakitimus'
  4. Nueiti i main/master saka - git checkout main/master
  5. Atnaujinti main/master saka - git pull
  6. Grizti i savo uzduoties saka - get checkout naujaSaka
  7. Sujungti savo saka su main/master saka ir isspresti galimai atsiradusius konfliktus - git merge
  8. Nusiunti savo darbines sakos pakitimus i gitHub'a - git push
  9. GitHub'e sukurti Pull Request'a
  10. Kiti komandos nariai patvirtina arba atmeta Pull Request'a
  11. Jeigu Pull Request'as sulauke pakankamai patvirtinimu - vykdyti merge'a
